Условие задачи:
Имеется n работ и m станков. Необходимо выполнить все работы на всех станках. Станок одновременно может выполнять только одну работу. И любая работа может обслуживаться только на одном станке. Задача рассматривается в среде плотных расписаний, т.е. никакая станок не может простаивать, если есть работа готовая к выполнению на нем.

Алгоритм:

Прежде всего выберем начальное рекордное решение. Построим его следующим образом. На каждом шаге для каждой работы будем из множества приборов, на которых данная работа еще не обслуживалась выбирать тот, время выполнения на котором для данной работы минимально.
Далее будем перебирать все возможные варианты, используя отсечение по рекорду и допустимости(используем поиск с возвратом).
Дерево перебора вариантов решения для данной задачи будет иметь следующий вид. Вершинам дерева соответствуют расписания некоторого множества работ и каждой работы на некотором множестве приборов. Корень дерева – нулевое расписание(ни один прибор не назначен ни на одну работу). Листьям дерева будут соответствовать расписания прохождения всеми работами всех приборов.
Частичное решение для вершины дерева x – это расписание прохождения некоторых работ на некоторых приборах. Значение целевой функции на частичном решении (F(x))� определяется как время выполнения расписания в x (F(корень)=0).
Если для некоторой вершины x F(x) > рекорда, то мы исключаем из рассмотрения вершину x и прекращаем ветвления из нее. Также используем отсечение по допустимости: т.к. мы строим порядок прохождения работами приборов в среде плотных расписаний, то мы будем исключать из рассмотрения те варианты, когда некотрый прибор свободен и есть работа готовая к выполнению на нем, но мы не назначаем эту работу на этот прибор, при отсутствии других подходящих приборов. Оптимальное расписание будет находиться в листе дерева.
Как только мы построили плотное расписание(дошли до листа дерева t), мы сравниваем рекорд и F(t) и выбираем рекорд=min{рекорд, F(t)}.
